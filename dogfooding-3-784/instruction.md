### 角色设定
你是一位资深的代码分析工程师，专注代码理解与分析。你精通多种编程语言和框架，能够从功能、安全、性能、架构等多个维度对代码进行深度分析。

### 项目目标
对给定的代码进行全面的理解与分析，覆盖 7 个核心维度，并输出结构化的分析报告。

### 分析对象
请读取 `environment/repo/` 目录下的所有代码文件，包括：
- `app/main.py` — FastAPI 应用入口
- `app/config.py` — 配置管理
- `app/models.py` — 数据模型定义
- `app/routers/analyze.py` — 分析任务路由
- `app/routers/batch.py` — 批量评测路由
- `app/routers/test_cases.py` — 测试用例路由
- `app/services/llm_client.py` — 大模型 API 调用
- `app/services/prompt_builder.py` — 提示词构建器
- `app/services/evaluator.py` — 评分引擎
- `app/services/test_case_store.py` — 测试用例库

请对以上所有文件进行全面分析。

### 分析范围

#### 1. 代码功能理解
- 准确概括代码核心功能（非逐行翻译）
- 识别所有输入参数及其类型
- 描述返回值类型和含义
- 识别副作用（I/O、全局状态修改、网络调用等）
- 说明核心算法或设计模式

#### 2. Bug 检测
- 逻辑错误（off-by-one、条件判断错误）
- 边界问题（空值、溢出、越界）
- 并发缺陷（竞态条件、死锁）
- 资源泄漏（未关闭连接、内存泄漏）
- 类型错误（隐式转换、类型不匹配）

#### 3. 复杂度分析
- 时间复杂度（最好/最坏/平均情况）
- 空间复杂度
- 递归深度分析
- 隐藏复杂度识别（字符串操作、哈希冲突等）
- 优化方案及其复杂度对比

#### 4. 代码重构建议
- 代码坏味道识别（过长函数、重复代码、数据泥团等）
- SOLID 原则检查
- 设计模式应用建议
- 功能等价的重构后完整代码
- 可读性/可维护性/可测试性改进对比

#### 5. 安全漏洞识别
- SQL 注入 / NoSQL 注入
- XSS（反射型/存储型/DOM 型）
- 命令注入 / 代码注入
- 路径遍历 / 文件包含
- 硬编码密钥 / 敏感信息泄露
- CSRF / SSRF / 反序列化漏洞

#### 6. 代码执行推演
- 逐步追踪变量状态
- 函数调用栈跟踪
- 递归展开过程
- 闭包和作用域分析
- 最终输出结果（必须 100% 准确）

#### 7. 跨语言翻译
- 保持功能完全等价
- 使用目标语言惯用写法（idiomatic）
- 类型系统差异处理
- 错误处理机制转换
- 语言间行为差异说明

### 约束类型

#### 1. 非代码回复约束
- 使用 Markdown 格式输出分析报告
- 代码块必须标注语言类型
- 复杂度使用标准大 O 表示法
- Bug 检测必须说明触发条件和影响范围
- 安全漏洞必须给出具体攻击向量描述
- 执行推演必须使用表格展示步骤和变量状态
- 不使用模糊词汇（如"可能"、"也许"、"大概"）

#### 2. 业务逻辑约束
- 功能理解必须准确抓住核心意图，非逐行翻译
- Bug 检测必须找出所有已知问题，不允许误报
- 执行推演最终输出必须 100% 准确
- 复杂度推导过程必须严谨，结论正确
- 重构后代码必须功能等价且完整可运行
- 翻译后代码必须使用目标语言惯用写法
- 不编造不存在的 API 或库

#### 3. 代码风格或规范约束
- 重构后代码结构清晰、职责分明
- 翻译后代码符合目标语言命名惯例
- 包含中文注释
