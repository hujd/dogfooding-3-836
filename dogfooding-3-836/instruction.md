### 角色设定
你是一位资深的前端代码分析工程师，专注 React 应用的代码审查与分析。你精通 React 生态、TypeScript、状态管理、组件设计模式，能够从功能、架构、安全、性能等多个维度对前端代码进行深度分析。

### 项目目标
对给定的 React Web 前端项目进行全面的代码理解与分析，覆盖以下核心维度，并输出结构化的分析报告。

### 分析对象
请读取 `environment/repo/` 目录下的所有代码文件，包括：
- `src/main.tsx` — 应用入口
- `src/App.tsx` — 根组件（路由、导航、主题）
- `src/pages/AnalyzePage.tsx` — 代码分析页面
- `src/pages/BatchPage.tsx` — 批量评测页面
- `src/pages/HistoryPage.tsx` — 历史记录页面
- `src/services/api.ts` — API 请求封装
- `src/store/index.ts` — Zustand 状态管理
- `src/utils/constants.ts` — 常量定义
- `vite.config.ts` — Vite 配置

请对以上所有文件进行全面分析。

### 分析范围

#### 1. 代码功能理解
- 整体项目功能概述（应用做什么）
- 各页面组件的功能说明
- 组件间数据流和通信方式
- 路由结构与页面导航逻辑
- 状态管理方案分析（Zustand store 结构）
- API 请求层封装方式

#### 2. Bug 检测
- React 组件生命周期问题（useEffect 依赖遗漏、闭包陷阱）
- TypeScript 类型安全问题
- 状态更新竞态条件
- 事件处理器内存泄漏（未清理的订阅/定时器）
- 条件渲染逻辑错误
- 边界情况处理（空数据、网络错误、加载状态）

#### 3. 复杂度分析
- 组件渲染复杂度（嵌套深度、重渲染频率）
- 状态管理复杂度
- 数据流复杂度（props drilling、Context 层级）
- 列表渲染性能（key 使用、虚拟化需求）

#### 4. 代码重构建议
- 组件拆分建议（过大组件、职责不单一）
- 自定义 Hook 提取机会
- 重复代码消除
- TypeScript 类型优化
- 样式组织改进

#### 5. 安全漏洞识别
- XSS 风险（dangerouslySetInnerHTML、用户输入渲染）
- 敏感信息泄露（API Key、Token 暴露在前端）
- CSRF 防护缺失
- 不安全的第三方依赖
- localStorage 敏感数据存储风险

#### 6. 代码执行推演
- 组件挂载和渲染顺序
- useEffect 执行时机和顺序
- 状态更新后的重渲染过程
- 用户交互事件的完整处理链路

#### 7. 架构评审
- 项目目录结构合理性
- 组件分层是否清晰（pages / components / hooks / services）
- 关注点分离是否到位
- 可扩展性和可维护性评估
- 与后端 API 的对接方式评估

### 约束类型

#### 1. 非代码回复约束
- 使用 Markdown 格式输出分析报告
- 按维度组织分析结果，每个维度独立章节
- Bug 检测必须指出具体文件和代码位置
- 安全漏洞必须给出具体攻击场景
- 重构建议必须给出修改前后的代码对比

#### 2. 业务逻辑约束
- 功能理解必须准确描述应用的核心用途和工作流
- Bug 检测必须基于 React 最佳实践，不误报正常用法
- 重构建议必须保持功能等价
- 架构评审必须结合项目实际规模给出合理建议，不过度设计

#### 3. 代码风格或规范约束
- 分析结论必须有代码引用支撑
- 包含中文注释和说明

### 输出要求
| 文件 | 格式 | 说明 |
|------|------|------|
| analysis_report.md | Markdown | 完整的前端代码分析报告，覆盖所有涉及的维度 |
| refactored_code/ | 目录 | 重构后的组件代码（如涉及重构维度） |
